5. HOW THE SYSTEM WORKS
================================================================================

REQUEST FLOW:

User Frontend
    ↓ (HTTP POST)
CareerGuidanceController
    ↓ (Calls)
CareerGuidanceService.getCareerGuidance()
    ↓ (Uses)
AIProviderService.generateContent(prompt)
    ↓
    ├─ TRY: GroqHttpClient.generate(prompt)
    │  ├─ Success (200) → Return Groq response ✓
    │  │  Log: "✓ Groq API succeeded - using GROQ provider"
    │  │
    │  └─ Failure or 429 → Continue to fallback
    │     Log: "⚠ Groq API rate limit exceeded (429). Switching to Gemini API..."
    │
    ├─ FALLBACK: GeminiHttpClient.generate(prompt)
    │  ├─ Success (200) → Return Gemini response ✓
    │  │  Log: "✓ Gemini API succeeded - using GEMINI provider (fallback)"
    │  │
    │  └─ Failure → Throw exception ✗
    │     Log: "✗ Gemini API also failed"
    │
Frontend ← Response (from Groq or Gemini)

6. KEY FEATURES IMPLEMENTED
================================================================================

✅ AUTOMATIC PROVIDER SELECTION:
   - Primary: Groq API (preferred for speed and cost)
   - Fallback: Gemini API (unlimited requests)

✅ RATE LIMIT HANDLING:
   - Detects 429 status code from Groq
   - Identifies "rate limit" error messages
   - Automatically switches to Gemini
   - No user-facing errors

✅ TRANSPARENT SWITCHING:
   - Frontend unaware of provider change
   - Same response format from both providers
   - Seamless experience for users

✅ COMPREHENSIVE LOGGING:
   - Logs which provider is being used
   - Tracks fallback events
   - Helps monitor usage patterns
   - DEBUG level logging for troubleshooting

✅ ERROR HANDLING:
   - Handles timeouts gracefully
   - Retry logic with exponential backoff
   - Proper error propagation if both fail
   - Detailed error messages

✅ CONFIGURABLE BEHAVIOR:
   - Both providers configurable in properties
   - Timeouts adjustable
   - Max retries configurable
   - Temperature and token limits customizable

11. WHAT HAPPENS NOW
================================================================================

SCENARIO 1: Within Groq Daily Limit
├─ User makes career guidance request
├─ System tries Groq API
├─ Groq responds with 200 OK
├─ Result: ✅ Career guidance returned
└─ Log: "✓ Groq API succeeded - using GROQ provider"

SCENARIO 2: Groq Daily Limit Exceeded (429)
├─ User makes career guidance request
├─ System tries Groq API
├─ Groq responds with 429 (rate limit)
├─ System automatically retries with Gemini
├─ Gemini responds with 200 OK
├─ Result: ✅ Career guidance returned from Gemini
└─ Log: "⚠ Groq API rate limit exceeded (429). Switching to Gemini API..."
       "✓ Gemini API succeeded - using GEMINI provider (fallback)"

SCENARIO 3: Both APIs Fail
├─ User makes career guidance request
├─ System tries Groq API → Fails
├─ System tries Gemini API → Fails
├─ Result: ❌ Error response
└─ Log: "✗ Gemini API also failed"
       "All AI providers failed. Groq and Gemini both unavailable."
