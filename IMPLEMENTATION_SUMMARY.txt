================================================================================
HOW IT WORKS
================================================================================

Request Flow:

User Request
    ↓
CareerGuidanceService
    ↓
AIProviderService.generateContent()
    ↓
    ├─ Try Groq API
    │  ├─ Success? → Return Groq response ✅
    │  └─ Rate Limit (429)? → Continue to fallback
    │
    ├─ Try Gemini API
    │  ├─ Success? → Return Gemini response ✅
    │  └─ Failure? → Return error ❌

Result: Career guidance ALWAYS available (unless both providers fail)

================================================================================
WHAT HAPPENS WHEN GROQ HITS RATE LIMIT
================================================================================

BEFORE (without fallback):
User Request → Groq Rate Limit (429) → ERROR to user ❌

AFTER (with fallback):
User Request → Groq Rate Limit (429) → Automatically try Gemini → SUCCESS ✅
User gets response, never knows about the fallback!
