4. HOW IT WORKS
================================================================================

FLOW DIAGRAM:

User Request
    ↓
CareerGuidanceService
    ↓
AIProviderService.generateContent(prompt)
    ↓
Try Groq API
    ├─ Success → Return response ✓
    └─ Failure or Rate Limit (429)
         ↓
      Try Gemini API
      ├─ Success → Return response ✓
      └─ Failure → Throw error ✗

EXAMPLE SCENARIO:

Day 1: User makes career guidance request
├─ Groq is used (tokens: 1/100000)
├─ Log: "✓ Groq API succeeded - using GROQ provider"
└─ Response returned to user

Day 5: User makes another request
├─ Groq is used (tokens: 95000/100000)
├─ Log: "✓ Groq API succeeded - using GROQ provider"
└─ Response returned to user

Day 5 (Evening): Another user request triggers 429
├─ Groq fails with 429 (rate limit exceeded)
├─ Log: "⚠ Groq API rate limit exceeded (429). Switching to Gemini API..."
├─ Gemini API is called
├─ Log: "✓ Gemini API succeeded - using GEMINI provider (fallback)"
└─ Response returned to user

Next Day: Groq limit resets
├─ First request uses Gemini (waiting for reset to complete)
├─ Once reset verified, back to using Groq
└─ Cycle continues

10. TROUBLESHOOTING
================================================================================

ISSUE: "Groq rate limit exceeded (429)"
SOLUTION: 
- This is normal when daily limit is reached
- System automatically switches to Gemini
- Wait for midnight UTC for Groq limit to reset
- Or upgrade to paid Groq tier

ISSUE: "Gemini API also failed"
SOLUTION:
- Check if Gemini API key is correct in application.properties
- Verify internet connection
- Check if Gemini API is not rate limited itself
- See https://aistudio.google.com for status

ISSUE: Both providers fail
SOLUTION:
- Verify both API keys are correct
- Check application logs for specific error
- Test API keys manually using curl:
  
  For Groq:
  curl -X POST "https://api.groq.com/openai/v1/chat/completions" \
    -H "Authorization: Bearer YOUR_GROQ_KEY" \
    -H "Content-Type: application/json" \
    -d '{"model":"llama-3.3-70b-versatile","messages":[{"role":"user","content":"Test"}]}'
  
  For Gemini:
  curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_GEMINI_KEY" \
    -H "Content-Type: application/json" \
    -d '{"contents":[{"parts":[{"text":"Test"}]}]}'

ISSUE: "Failed to build request body"
SOLUTION:
- Check prompt length (shouldn't exceed ~30k characters)
- Verify input is valid JSON if passing structured data
- Check for special characters in prompt
