HOW IT WORKS
================================================================================

BASIC FLOW:

User makes request
    ↓
AIProviderService checks provider health
    ↓
IF Gemini healthy:
    Try Gemini
    ├─ Success? → Return from Gemini ✅
    └─ Any error? → Mark Gemini unhealthy, try Groq
        ↓
    IF Groq healthy:
        Try Groq
        ├─ Success? → Return from Groq ✅
        └─ Any error? → Mark Groq unhealthy, fail ❌
    ELSE:
        Both providers unavailable → Return error

ELSE (Gemini already unhealthy):
    Skip Gemini, directly try Groq
    ├─ Success? → Return from Groq ✅
    └─ Fail? → Both unhealthy → Return error

REQUEST FLOW EXAMPLE
================================================================================

SCENARIO: User clicks "Mock Interview" → Generate Question

Request arrives:
  User clicks "Start Mock Interview"
  Frontend calls: POST /api/ai/mock-interview/generate-question

Backend processes:
  1. Request received
  2. Logs: "Provider Health - Gemini: ✅ OK | Groq: ✅ OK"
  3. Attempts Gemini:
     ├─ Sends prompt to Gemini API
     ├─ Receives response
     ├─ Logs: "✅ [SUCCESS] Gemini API succeeded"
     └─ Returns response to frontend
  4. Frontend displays question ✅

SCENARIO: Gemini fails (e.g., rate limit)

Request arrives:
  User clicks "Generate Question"

Backend processes:
  1. Logs: "Provider Health - Gemini: ❌ DOWN | Groq: ✅ OK"
  2. Attempts Gemini anyway (attempts configured):
     ├─ Sends prompt
     ├─ Receives 429 error
     ├─ Logs: "❌ [PRIMARY ERROR] Gemini failed: 429 Rate limit"
     └─ Marks geminiHealthy = false
  3. Falls back to Groq:
     ├─ Sends same prompt to Groq
     ├─ Receives response
     ├─ Logs: "✅ [SUCCESS] Groq API succeeded ✓ USING GROQ PROVIDER"
     └─ Returns response to frontend
  4. Frontend displays question ✅ (from Groq, not Gemini)

SCENARIO: Both providers fail

Request arrives:
  User clicks "Generate Question"

Backend processes:
  1. Attempts Gemini → FAILS with network error
  2. Marks geminiHealthy = false
  3. Attempts Groq → FAILS with API key error
  4. Marks groqHealthy = false
  5. Logs: "❌❌❌ [CRITICAL] Both AI providers failed!"
  6. Throws exception
  7. Frontend receives error message:
     "All AI providers currently unavailable"
