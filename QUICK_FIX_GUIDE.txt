================================================================================
OPTIMIZATION TIPS
================================================================================

To avoid hitting Groq limit too quickly:

1. Reduce max tokens in application.properties:
   groq.max-output-tokens=1024  (was 2048)

2. Cache responses to avoid duplicate requests

3. Monitor your usage:
   Check logs daily for "Switching to Gemini" messages
   If frequent, consider:
   - Reducing output tokens more
   - Upgrading to paid Groq
   - Increasing usage of Gemini

================================================================================
IF SOMETHING GOES WRONG
================================================================================

1. Backend won't compile?
   mvn clean compile -DskipTests -q
   If still fails, check:
   - Java version: 17+ required
   - Maven installed and working
   - No syntax errors in code

2. Career Guidance returns error?
   Check logs for:
   - "Groq authentication failed" → Wrong Groq API key
   - "Gemini authentication failed" → Wrong Gemini API key
   - "Network error" → Internet connectivity issue
   - Replace API keys in application.properties

3. Always using Gemini (never Groq)?
   Check logs for reason
   Likely: Groq API key is wrong
   Fix: Update groq.api-key in application.properties

4. Groq and Gemini both fail?
   Verify both API keys are correct
   Test manually using provided curl commands
   See GROQ_RATE_LIMIT_SOLUTION.txt for details
